### 空格与 Tab

Tab 是制表符，其作用是将光标位置调整为下一个 8 的倍数（或者在大部分 IDE 里是 4 的倍数）。

也就是说，其相当于一段空白文本，长度等于到下一个 4 的倍数的位置的距离。

Python 的缩进理论上只考虑缩进使用的字符是否一致，所以用 Tab 或者 4 个空格，甚至任意多个空格都可以。

PEP8 规范要求统一使用 4 个空格或者 Tab，并且考虑到 Tab 在不同地方显示不同（有些地方时 4 空格有些地方是 8 空格），因此强烈建议统一使用 4 空格进行缩进。

在大部分主流 IDE 都会自动用空格代替 Tab（即按下 Tab 后将其转化为相应的空格数量）。

你可以尝试在 IDLE 中使用 Tab 进行缩进，看看是不是转化为了空格。

## if-elif-else 语句

我们刚刚在嵌套语句中举了一个在 `if{:python}` 语句中使用 `if{:python}` 的例子。

这种语句实际上很不友好，既不符合人类思考习惯，也容易造成缩进地狱，比如：

```python
name = 'Alice'

if name == 'John':
    print('Hello, John')
else:
    if name == 'Bob':
        print('Hello, Bob')
    else:
        if name == 'Jack':
            print('Hello, Jack')
        else:
            if name == 'Alice':
                print('Hello, Alice')
            else:
                print('Hello')
```

本来看起来平级却出现在了不同的缩进下。

为了解决这个问题，Python 引入了 `elif{:python}` 语句，将上面的代码改为：

```python
name = 'Alice'

if name == 'John':
    print('Hello, John')
elif name == 'Bob':
    print('Hello, Bob')
elif name == 'Jack':
    print('Hello, Jack')
elif name == 'Alice':
    print('Hello, Alice')
else:
    print('Hello')
```

同样地，最后的 `else{:python}` 语句可以省略。

## 布尔类型

说到条件控制，就不得不提对应的布尔类型了。

与其他类型不同，布尔类型只有两个取值：`True{:python}` 和 `False{:python}`。

刚才所有 `if` 语句中的条件实际上就是一个计算结果为布尔类型的表达式。

### 数学表达式

最常见的条件就是数学中的判断运算，比如大小于，这些运算都很容易在 Python 中实现。

特别地，等于是 `==`，不等于是 `!=`。

### 逻辑表达式

如果不能用单个数学表达式描述条件，那么可能就需要逻辑表达式来帮忙了。

Python 支持两个二元逻辑表达式：`and` 和 `or`，以及一个一元逻辑表达式 `not`。

<Sandbox title="使用逻辑判断" value={use_and_or_statement} />

### 保存到变量

布尔值也是一种数据，你可以将其保存到变量：

```python
cond = 1 == 2

if cond:
    print('The statement is true.')
else:
    print('The statement is false.')
```

你甚至可以输出布尔值：

<Sandbox title="输出布尔值" value={"print('1 == 2 is', 1 == 2)\n"} />

这里新手容易犯的一个错误是，即使已经得到了布尔值，依然尝试用 `==` 作判断：

```python
cond = 1 == 2

if cond == True:
    print('The statement is true.')
else:
    print('The statement is false.')
```

这样做不会有问题，但是完全没有必要。
