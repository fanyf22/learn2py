# 代码规范与 PEP

虽然本节的知识和 Python 语法无关，但是对之后的编程有一些帮助。

## 代码规范

语法是代码必须满足的要求，而规范也是强烈建议满足的要求。

使用统一的规范写代码可以让代码可读性更强，别人阅读时也能更流畅。

Python 官方提出了一个标准的代码规范，所有 Python 使用者**强烈建议**使用这一套规范。

这里列举一些已经学过的知识相关的代码规范：

* **字符串**：字符串可以用单引号或双引号括起来，但是要求统一使用其中之一，推荐使用单引号。
* **变量**：变量名要按照其含义的英语命名，使用小写名称并且用 `_` 连接单词，例如 `calc_result`。
* **赋值与运算**：在运算符、`=` 两边都应该加上空格，例如 `x = 1 + 2`。
* **函数参数**：函数参数用 `,` 隔开，在 `,` 的后面要加上空格，例如 `print('Hello', 'world')`。
* **最大字符数**：一行的字符数不应该超过 79 个，为了保证代码不超出可视区域且有良好的视觉效果。

实际上，大部分主流 IDE 也会根据这一官方代码规范进行代码检查，所以强烈建议遵守。

在之后的教程中，每次出现新的语法都会讲解对应的代码规范。

## PEP

PEP，全名 Python Enhancement Protocols，中文可以翻译为“Python 增强提案”或者“Python 改进建议书”，由 Python 开发者和社区核心成员维护，用于提出、讨论和记录 Python 语言的新特性、改进和相关信息的正式文档。

PEP 的官网是 [](https://www.python.org/dev/peps)，每一条提案都有一个编号，有一些提案是已经在某个 Python 版本中被实现的，而一些最新的可能是对未来 Python 特性的设想。

刚才所讲的 Python 官方代码规范就对应了 PEP8，可以在官网自行查找阅读。
